# Projeto Limites Sistêmicos

Repositório destinado ao gerenciamento de artefatos de software, documentos técnicos e demais registros do projeto.

## Objetivos
- Centralizar a gestão de versões e artefatos digitais
- Garantir rastreabilidade das mudanças
- Apoiar as práticas de auditoria, governança e conformidade

## Estrutura do Repositório
- `/src` → código-fonte do projeto
- `/docs` → documentação e manuais
- `/tests` → scripts e casos de teste

## Convenções
- Branch principal: `main`
- Branches de desenvolvimento: `feature/nome-funcionalidade`
- Commits padronizados no formato: `[TIPO] - Descrição`  
  - Exemplo: `[FIX] - Correção na validação de input`

## Tecnologias
- Linguagem: Python / Java / (escolher a usada no projeto)
- Ferramenta de versionamento: Git
- Repositório: GitHub/GitLab/Bitbucket
